# MATTBEAR Repo Focus

Updated: 2026-06-21

## Purpose

`bearicide.github.io` is the public front door for the MATTBEAR ecosystem.

It should answer fast:

1. What is MATTBEAR?
2. What can people click first?
3. What is professional proof?
4. What is experimental/playable?
5. Where does each district belong?

No mystical fog machine required. We have enough of those in CSS already.

---

## Public route order

1. **Main**  
   The front door. Highest polish. Fastest paths.

2. **Examples**  
   Professional proof and client-facing showcase.

3. **Project Focus / Projects**  
   Route board for the ecosystem, not a junk drawer.

4. **HiberNation**  
   Community, shared projects, groups, archives, Discord energy.

5. **MATTBEARCADE / RETROCADE**  
   Playable game districts.

6. **Music / Music Toys / The Choppa**  
   Audio, releases, tools, samplers, MIDI, experiments.

7. **Lab / Archive**  
   Experimental pages, old shells, duplicates, seasonal sets.

---

## Active priorities

### 01. Shared shell
Every public page should use the same basic system:

- MATTBEAR brand link
- Main / HiberNation / Arcade / Examples / Music route order
- readable footer
- related-project links
- mobile-first spacing
- no dead-end pages

### 02. Project clarity
Every page needs to answer:

- What is this?
- Why does it exist?
- What do I click next?
- Is this live, prototype, archive, or lab?

### 03. Mobile cleanup
Priority mobile checks:

- header height
- button size
- no horizontal overflow
- readable card spacing
- no hidden nav traps
- touch targets large enough for actual human fingers, regrettably

### 04. Asset discipline
Use the existing shared visual language before generating new assets:

- obsidian / chrome / gold base
- cyan system
- magenta media
- green live
- amber alert
- red danger
- shared buttons, panels, HUD framing

### 05. Repo discipline
Do not create another repo unless it has a clear district reason.

Preferred order:

1. add page inside existing repo
2. add folder inside existing district
3. only then create a new repo
